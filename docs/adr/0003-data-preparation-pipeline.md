# ADR 0003: 数据准备流水线设计与格式转换架构

## 状态
已通过 (Accepted)

## 上下文背景
平台需要支持多种上游原始数据集格式（如 Pascal VOC、COCO、YOLO）的一键导入、统一过滤与智能切分。随着训练数据规模达到百万级，传统的一次性加载巨型 JSON（如 COCO）的设计极易引发 OOM（内存爆损），且缺乏数据合规审计及 Fail-Fast 熔断机制。

## 架构决策
1. **延迟加载驱动注册表**: 采用装饰器机制解耦格式解析驱动，上层通过统一的网关动态获取驱动，实现格式的横向平滑扩展。
2. **COCO 流式 tempfile 中转**: 针对 COCO 巨大 JSON 格式，利用操作系统级 `tempfile` 机制进行中间流式规整与持久化解耦，避免常驻内存撑爆系统。
3. **分层依赖注入（DI）与接口等价性**: `YoloConverter` 作为目标格式保持等价的解析接口；物理落地器 `DatasetMaterializer` 采用注入目标路径的设计，完全不与具体物理路径硬编码绑定，便于单测。
4. **元数据审计追踪**: 在最终产出的训练 YAML 配置文件中，特殊注入 `odp_meta` 审计块（包含 `schema_version`、随机种子及各子集样本精确数量），实现完整的数据线索追踪（Data Lineage）。
5. **Fail-Fast 安全熔断**: 数据流水线启动前执行覆盖率合规检查，低于 50% 的脏数据集立即拒绝处理，最大化节约算力。

## 后果影响
- **正面**: 成功将上游数据格式与核心切分业务完全剥离；多进程并发转换时不会发生路径读写冲突；具备完备的生产级防腐与可复现性。
- **负面**: 数据物理落盘时发生了一次磁盘拷贝，在大数据集下会占用一定的额外存储空间。
